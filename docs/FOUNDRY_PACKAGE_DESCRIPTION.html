D35E Piecemeal Armor And Called Shots

D35E Piecemeal Armor And Called Shots adds RAW-adapted optional-rule helpers for D35E worlds. It supports two table-facing workflows: configurable piecemeal armor components and called-shot handling inside D35E's native attack and Apply Damage flow.

This module is an adaptation aid for D&D 3.5e tables. It does not claim that piecemeal armor or called shots are official D&D 3.5 RAW. The bundled defaults are adapted from Pathfinder 1e Ultimate Combat optional rules where D35E can support them. A legacy v1.0 workflow is available for tables that prefer manual outcomes.

Support: If this module helps your table, donations are optional and support continued maintenance, compatibility testing, release packaging, and documentation.

Install

In Foundry, open Add-on Modules > Install Module, paste this into Manifest URL, and install:

https://github.com/SpencerZPoole/d35e-piecemeal-armor-called-shots/releases/latest/download/module.json

After installation, enable D35E Piecemeal Armor And Called Shots in a D35E world and reload when Foundry asks.

What It Adds

  • Marks D35E equipment as piecemeal armor components.
  • Provides a starter catalog for common RAW armor pieces such as padded, leather, chain, and plate arms/legs/torso.
  • Previews, syncs, and restores one D35E-native aggregate armor item using RAW-adapted piecemeal armor formulas.
  • Keeps synced components visible as reversible visual records while preventing duplicate armor weight.
  • Adds a Called Shot dropdown inside D35E's native attack/use dialog.
  • Injects called-shot penalties into D35E attack math, so expanded rolls show entries such as Called Shot: Ear -10.
  • Can adjust D35E's native Apply Damage AC check for called shots that target weaker or stronger piecemeal armor locations.
  • Enforces Improved/Greater Called Shot full-attack limits in RAW-adapted mode, with legacy permissive policies still available.
  • Automatically applies called-shot severity outcomes after D35E Apply Damage and records a GM restore ledger.
  • Includes an in-Foundry profile editor for locations, penalties, coverage slot(s), and outcome effects.

Piecemeal Armor Workflow

Equipment items can be configured as armor components with module-owned flags. Piece category drives the arms/legs/torso formula, while coverage fields can name one or more called-shot locations, such as head; eyes; ears or torso, arms, legs. When the actor is ready, the GM can preview the combined profile, sync one aggregate D35E armor item, and restore component armor fields later if needed.

The module uses D35E-native item data for the final armor contribution. Components remain visible as component records, while the aggregate item is the one intended to contribute armor math. The aggregate records the total component weight for transparency but contributes zero item weight so encumbrance is not counted twice.

Called-Shot Workflow

Players use the normal D35E attack or item controls. The native attack dialog gains a Called Shot selector. Choosing a location applies the configured attack penalty through D35E's existing roll breakdown and posts a GM-facing outcome card after the roll.

RAW-adapted full attacks honor Improved Called Shot and Greater Called Shot. Legacy mode can ask per attack, apply to the first attack only, apply to every attack, or ignore called-shot selections, depending on module settings.

Automatic Outcomes And Restore

Local armor AC can change D35E's native Apply Damage hit and crit check when enabled. In RAW-adapted mode, D35E Apply Damage determines hit, crit, and post-DR damage; the module then determines normal, critical, or debilitating severity and applies the configured outcomes. Severe outcomes such as death or permanent maiming are recorded in a called-shot ledger so a GM can restore them from the target actor sheet if needed.

Configuration

Module settings live under Game Settings > D35E Piecemeal Armor And Called Shots. GMs can choose RAW-adapted or legacy rules mode, enable or disable each workflow, choose full-attack handling, enable local armor AC for Apply Damage, show location armor overlays, and edit called-shot profiles.

Profiles control location labels, attack penalties, enabled state, matching armor coverage slot(s), and outcome effects. Effect snippets are JSON because they map directly to the module's declarative effect engine.

Boundaries

The module is additive. It does not edit D35E system files, mutate actors on world load, or enable optional automation without the world enabling the module.
