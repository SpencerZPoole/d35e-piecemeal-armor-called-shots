# D35E Piecemeal Armor And Called Shots v1.0.1

This release is a live-audited polish and reliability update for the first public package.

## Fixes

- Generated piecemeal armor aggregate items are explicitly equipped after D35E creates or updates them, so the aggregate item is ready to contribute native armor math.
- Syncing with no equipped piecemeal armor pieces now safely returns without creating a zero-value aggregate armor item.
- Native attack-form parsing now handles real HTML form elements correctly, preserving called-shot selection and per-attack full-attack queues.
- The armor sync dialog now explains when no syncable pieces are available and only offers actions that make sense for the actor's current state.

## Polish

- Called-shot chat outcome buttons now use clear title-case labels and accessibility text.
- Invalid called-shot profile JSON imports now report a friendly error instead of failing silently in the profile editor.
- Public README, user guide, Foundry package description, release process, and package-listing packet were refreshed for the current release.

## Compatibility

- Foundry VTT: verified on 14.362.
- D35E system: verified on 3.0.2.
- Foundry v13 is not marked verified yet.

## Rules Boundary

The bundled called-shot and piecemeal armor defaults are PF1e-derived optional-rule scaffolding. They are configurable and are not represented as official D&D 3.5 RAW.
