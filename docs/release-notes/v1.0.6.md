# D35E Piecemeal Armor And Called Shots v1.0.6

This patch fixes two user-reported piecemeal armor UX issues and improves local armor matching for called shots.

## Fixed

- Synced armor components now remain visible as reversible `misc`/`clothing` visual records in sensible equipment slots instead of being forced to `slotless`.
- The generated aggregate armor item now contributes `0` item weight so encumbrance is not counted twice.
- The aggregate still records the combined component weight in its module summary and preview.
- Existing v1.0.5 synced components reuse their native backup data on the next Sync or Restore.

## Improved

- Component and profile coverage fields now support multiple locations separated by commas, semicolons, pipes, slashes, or line breaks.
- Local armor AC matching now treats coverage like `head; eyes; ears` as a match for Head, Eye, or Ear called shots.
- Multi-location components are counted once even when more than one listed coverage value overlaps the called-shot location.
- Item-sheet and profile-editor labels now say `Coverage slot(s)` with examples.

## Compatibility

- Foundry VTT: verified on 14.362.
- D35E system: verified on 3.0.2.
