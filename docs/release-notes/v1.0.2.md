# D35E Piecemeal Armor And Called Shots v1.0.2

This release fixes the first public piecemeal armor support report.

## Fixes

- The `Piecemeal Armor` item fieldset now appears inside the D35E equipment sheet `Details` tab instead of being appended outside the visible tab layout.
- Piecemeal armor components are now module-managed by default. A checked component no longer has to occupy a D35E armor slot before sync.
- Miscellaneous equipment can now act as an armor component by storing armor values in the module fieldset.
- Inventory badges now appear only on marked pieces, synced components, and generated aggregate items.

## Workflow Clarification

Component items may be armor, shield, or miscellaneous equipment records. Use the module fieldset for armor-piece values and coverage. The generated `Piecemeal Armor Aggregate` item is the item that should contribute D35E armor AC after sync.

## Compatibility

- Foundry VTT: verified on 14.362.
- D35E system: verified on 3.0.2.

## Rules Boundary

The bundled called-shot and piecemeal armor defaults are PF1e-derived optional-rule scaffolding. They are configurable and are not represented as official D&D 3.5 RAW.
